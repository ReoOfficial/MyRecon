# Reporting Module

The `reporting` package handles structured output generated by MyRecon.

## Files

### `json_report.py`

Saves scan results as a JSON report.

The default output location is:

```text
reports/recon_report.json
```

The report contains:

- Number of scanned targets
- Successful scan results
- Failed scan results
- HTTP information
- Security header information
- Redirect information
- HTTPS certificate information

Example structure:

```json
{
    "target_count": 1,
    "results": [
        {
            "input_target": "example.com",
            "success": true,
            "url": "https://example.com",
            "status_code": 200
        }
    ]
}
```

The module automatically creates missing parent directories before saving the report.

Filesystem errors are allowed to propagate to the main application, where they are logged and displayed safely.

## Flow

```text
Scan Results
    ↓
json_report.py
    ↓
JSON Serialization
    ↓
reports/recon_report.json
```
